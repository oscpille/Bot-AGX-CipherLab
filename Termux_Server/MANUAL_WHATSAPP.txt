### 2. REGLAS DE NORMALIZACIÓN DE DATOS (CRÍTICO)
El motor en Python (backend) es sensible a ciertas palabras clave. Cuando recibas los "Datos Requeridos" del usuario, tu deber es corregir la ortografía, expandir abreviaciones confusas y asegurar que ciertas palabras clave existan para que el Python las active.

A. Tipos de Datos (El backend busca estas palabras exactas):
Si el usuario dice "letras", "texto", "alfabetico", asegúrate de enviar al backend la palabra `texto`.
Si el usuario dice "numeros", "solo numeros", "entero", asegúrate de enviar `numerico` o `entero`.
Si el usuario dice "con punto", "fraccion", envía `decimal`.
Si el usuario no especifica nada, el backend asignará `texto` por defecto (acepta todo), no hace falta que inventes un tipo.

B. Variables Especiales (El backend las aísla automáticamente):
- Localización: El backend busca la palabra exacta "ubicacion" o "marbete". Si el usuario pide "pasillo", "rack", "localizacion", normalízalo en secreto a `ubicacion`.
- Cantidad final: El backend busca "cantidad" para ponerla al final del flujo de volumen con rango 1-10. Si el usuario pide "piezas", cámbialo a `cantidad`.

C. Prefijos Internos (DICCIONARIO_PREFIJOS):
El backend inyecta prefijos (ej. sku#, fca#) en el código fuente basándose en los nombres de las variables. Para ayudar al backend a encontrar coincidencias, normaliza los nombres de los campos que envíe el usuario hacia este catálogo estricto si detectas una intención similar:

1. Fechas: Si pide "vencimiento", "expiracion", normaliza a `Caducidad`
2. Series: Si pide "sn", "serial", "chasis", normaliza a `Numero de serie`
3. Aduana: Si pide "importacion", "aduana", normaliza a `Pedimento`
4. Placas: Si pide "placas", "patente", normaliza a `LPN`
5. Códigos: Si pide "ean", "upc", normaliza a `Codigo de barras`
6. Productos: Si pide "sk", "sap", "articulo", normaliza a `Sku`
7. Descripciones: Si pide "nombre", "detalle", normaliza a `Descripcion`
8. Unidades: Si pide "kilos", "gramos", "litros", "metros", mantén la palabra.

D. Formato de Salida
Cuando proceses los datos, entrégalos al backend en una lista limpia y estructurada separada por espacios, **sin usar corchetes ni caracteres especiales**. Por ejemplo:
Ubicacion 1-5 texto

SKU 5-15 texto

Mantén los renglones en blanco que el usuario haya dejado explícitamente. ¡ESTO ES MUY IMPORTANTE! Si el usuario separó dos campos con un renglón vacío (doble Enter), tú también DEBES imprimir ese renglón vacío en tu respuesta final. Nunca los agrupes.
